# 代码规则

## 代码结构与导入规范

- 项目始终使用别名导出而不是相对路径
- 禁止从子路径导入（如 `@/component/icons`、`@/component/theme`），必须使用 `@/component` 统一导入
- 导入时禁止使用文件后缀（如 `.tsx`、`.ts`）

## 懒加载规范

- pages 下的页面组件必须使用 React.lazy 懒加载
- 工具类组件（component 目录下的非页面组件）不使用懒加载

## 样式组织

- 组件样式使用对象管理，只有存在多种选择的变体（如 variant/size）才用变量存储，避免每次组件渲染重新生成对象
- 非对象的静态样式直接写在组件里，不需要提取到外部

## 组件属性组织

- 组件需要支持子元素属性透传时，必须使用统一的 itemProps 对象结构
- itemProps 必须是对象类型，包含所有需要透传的子元素属性
- 示例：itemProps = { trigger: {...}, title: {...}, content: {...}, nav: {...} }
- 禁止将子元素属性作为组件的顶级属性单独暴露
- 每个子元素属性必须使用对应的 HTML 元素类型进行类型约束
- 必须排除组件内部已处理的属性（如 href、onClick 等）

## 容器规范

- **禁止多余容器嵌套**：当父元素已是语义化容器（如 `<nav>`、`<aside>`、`<main>`）时，子元素不应再使用 `<div>` 包裹
- 应直接使用语义化标签包裹内容，如 `<nav><ul><li>...</li></ul></nav>` 而非 `<nav><div><ul>...</ul></div></nav>`
- 原因：避免不必要的 DOM 嵌套，保持结构简洁
