## Usage

<Callout type="info" title="Native Alternative for Chrome 140+">
  If you are targeting the latest Chrome (version 140 or above), the native
  `<select>` element with `multiple` and `size={1}` provides comparable UX with
  better performance out of the box. You can simply spread the props and
  override the attributes:
</Callout>

```tsx
<select {...props} multiple size={1}>
  {options.map((opt) => (
    <option key={opt.value} value={opt.value}>
      {opt.label}
    </option>
  ))}
</select>
```

## API Reference

### MultiSelect

A multi-select component that allows selecting multiple options.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `(SelectOption \| SelectOptionGroup)[]` | - | List of options or option groups |
| `value` | `string[]` | - | Controlled selected values |
| `defaultValue` | `string[]` | `[]` | Uncontrolled default selected values |
| `onChange` | `(values: string[]) => void` | - | Callback when selection changes |
| `placeholder` | `string` | - | Placeholder text |
| `invalid` | `boolean` | `false` | Whether the select is in invalid state |
| `className` | `ClassNameValue` | - | Accepts strings, arrays, and all falsy values (false, null, undefined, 0, 0n). True and objects are not supported. |

### SelectOption

| Prop | Type | Description |
|------|------|-------------|
| `label` | `string` | Display label for the option |
| `value` | `string` | Unique value for the option |

### SelectOptionGroup

| Prop | Type | Description |
|------|------|-------------|
| `group` | `string` | Group label |
| `options` | `SelectOption[]` | Options in this group |
