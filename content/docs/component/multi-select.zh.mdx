## 用法

<Callout type="info" title="Chrome 140+ 原生替代方案">
  如果你面向的是最新的 Chrome 浏览器（版本大于 140），原生 `<select>` 配合
  `multiple` 和 `size={1}` 即可获得相当的体验，且性能更好。直接展开 props
  并覆盖相关属性即可：
</Callout>

```tsx
<select {...props} multiple size={1}>
  {options.map((opt) => (
    <option key={opt.value} value={opt.value}>
      {opt.label}
    </option>
  ))}
</select>
```

## API 参考

### MultiSelect

支持选择多个选项的多选组件。

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `(SelectOption \| SelectOptionGroup)[]` | - | 选项列表或选项分组 |
| `value` | `string[]` | - | 受控的选中值 |
| `defaultValue` | `string[]` | `[]` | 非受控的默认选中值 |
| `onChange` | `(values: string[]) => void` | - | 选择变化时的回调函数 |
| `placeholder` | `string` | - | 占位文本 |
| `invalid` | `boolean` | `false` | 是否处于无效状态 |
| `className` | `ClassNameValue` | - | 自定义 CSS 类。仅支持字符串、数组及各类假值。 |

### SelectOption

| Prop | Type | Description |
|------|------|-------------|
| `label` | `string` | 选项的显示标签 |
| `value` | `string` | 选项的唯一值 |

### SelectOptionGroup

| Prop | Type | Description |
|------|------|-------------|
| `group` | `string` | 分组标签 |
| `options` | `SelectOption[]` | 该分组中的选项 |
